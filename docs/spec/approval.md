## 草案范围摘要

- IP 类型：小型单周期 RV32I CPU。
- 执行模型：在指令存储器和数据存储器均满足单周期响应假设时，每条受支持指令单周期完成。
- 接口假设：独立指令存储器读接口和数据存储器读写接口，基础接口不包含 ready/valid 或总线错误握手。
- 异常/错误处理：非法指令、不支持指令、地址未对齐、`ECALL`、`EBREAK` 进入 trap/halt 状态，等待复位恢复。
- 范围限制：不包含流水线、缓存、中断、特权模式、CSR、乘除法、压缩指令、原子指令或总线适配器。

## 已接受假设

以下关键假设作为本次批准基线的一部分接受：

1. 复位方式是否接受当前草案中的同步复位假设。
2. `RESET_PC = 32'h0000_0000` 是否为期望启动地址。
3. `trap`、`trap_cause`、`pc` 是否需要作为强制顶层端口。
4. `FENCE.I` 应作为非法指令 trap，还是作为 NOP 处理。
5. 是否需要加入任何 CSR、特权级、中断或 trap vector 支持。
6. 是否需要 ready/valid 存储器接口来支持非单周期存储器。
7. 是否存在目标频率、面积、FPGA/ASIC 工艺或地址宽度约束。
